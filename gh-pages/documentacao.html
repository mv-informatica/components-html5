Fornece um componente para realizar a paginação de grids, esse elemento pode realizar o envio por ajax ou submit.

Segue os atributos enviados ao servidor.

  • page - Informa a página que será listada na tabela;
  • size - Quantidade de elemento do page ou seja quantos elementos vão ser exibidos na tabela;
  • idToRender - Id do template do thymeleaf que será renderizado pelo servidor;

Segue um exemplo de como responder o componente de paginação do lado do servidor.

{% highlight java %} @RequestMapping(value="/listPaginated", method = RequestMethod.GET) public String findTipoFrequenciaPaginated(@ModelAttribute TipoFrequencia tipoFrequencia, @RequestParam(value = "page", defaultValue = "") Integer page, @RequestParam(value = "size", defaultValue = "") Integer size, @RequestParam(value = "idToRender", defaultValue = "") String idToRender, Model model, HttpServletRequest request) { Pageable pageable = new PageRequest(page, size, Sort.DEFAULT_DIRECTION, "descricaoFrequencia"); this.page = tipoFrequenciaBusiness.listTipoFrequencia(tipoFrequencia, pageable); model.addAttribute("page", this.page); model.addAttribute("tipoFrequencia", tipoFrequencia); if(isAjax(request)) return "tipoFrequencia/tipoFrequenciaList :: #" + idToRender; else return "tipoFrequencia/tipoFrequenciaList"; } private boolean isAjax(HttpServletRequest request){ String header = request.getHeader("x-requested-with"); if(header != null && header.equals("XMLHttpRequest")) return true; else return false; } {% endhighlight %}

Alerts

Proporciona serviço alerta aos usuários.

O componente foi criado usando a lib: Notify.

{% highlight html %}
{% endhighlight %}

Os alerts são usados como padrão no decorator que é herdado de todas as telas do tipo manter. Para usar basta informar o message no controller como no exemplo a seguir.

O EnumTipoMensagem possui as messagens padrões de uma tela do tipo manter.

{% highlight java %} class MyController { private final GenericMessages genericMessages; public String myMethod(){ //Determina uma menssagem de sucesso. model.addAttribute(EnumTipoMensagem.SUCESSO.getDescricao(), genericMessages.getMessage("messages.properties")); } } {% endhighlight %}

O Alert também pode ser usado diretamente do javaScript. Veja o exemplo a seguir.

{% highlight javaScript %} $.alertSuccess('msg de sucesso'); $.alertInfo('msg de informação'); $.alertError('msg de erro'); $.alertWarning('msg de perigo'); {% endhighlight %}

Também é possível usar combinado com requisições ajax.

Do lado do servidor basta lançar um GenericException. Veja o exemplo:

{% highlight java %} class MyController { public String myMethod() throws GenericException{ throw new GenericException("Este medicamento item já foi adicionado.", EnumTipoMensagem.ERRO); } } {% endhighlight %}

E no jquery basta usar como no exemplo abaixo.

{% highlight javaScript %} $.get( url, params ) .fail( function(response) { $.alertSuccess($.getGenericMessage(response)); } ); {% endhighlight %}
